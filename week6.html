Viikko 6

Cloudflare Web Analytics

Lyhyt pohdinta seurannan käytöstä ja CORS-ongelmien välttämisestä.

Mihin seurantaa voisi hyödyntää?

Cloudflare Web Analytics tarjoaa anonyymin tavan kerätä tietoa sivuston käytöstä ilman evästeitä tai henkilötietoja. Kerätyn datan avulla voi seurata sivulatauksia, kävijämääriä, suosituimpia sivuja sekä kävijöiden maantieteellistä jakautumista. Tätä tietoa voi hyödyntää sivuston teknisen toimivuuden valvonnassa: nähdään esimerkiksi onko jokin sivu hidas tai tuottaako se virheitä. Käyttäjäpolkujen analysointi paljastaa, mitkä sisällöt herättävät kiinnostusta ja mistä kävijät poistuvat. Kurssityössä tämä auttaa ymmärtämään, ovatko viikkotehtävät saavutettavissa ja löytyvätkö linkit etusivulta. Lisäksi mukautetuilla tapahtumilla (trackEvent) voi seurata yksittäisten toimintojen, kuten kirjautumisen tai pelin aloittamisen, onnistumista. Koska data on anonyymia, GDPR-vaatimukset täyttyvät helpommin, mutta käyttäjältä kysytään silti suostumus ConsentBanner-komponentilla.

CORS-ongelman välttäminen

CORS-virhe syntyy, kun selain estää eri verkkotunnukselta haetun resurssin käytön. Sen voi välttää tarjoamalla rajapinnasta oikeat Access-Control-Allow-Origin-otsakkeet, käyttämällä samaa verkkotunnusta sekä etu- että taustapäälle (esim. Cloudflare Pages + Workers), tai kierrättämällä pyynnöt oman taustapalvelimen kautta proxyna. Cloudflare Web Analyticsin beacon ei aiheuta CORS-ongelmaa, koska se lähettää tiedot yksisuuntaisena kuva-/POST-pyyntönä eikä lue vastausta JavaScriptillä.
